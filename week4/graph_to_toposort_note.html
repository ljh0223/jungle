Week 4 Learning Note

그래프부터 위상정렬까지

정의만 외우지 않고, 문제를 풀 때 바로 떠올릴 수 있게 정리한 재혁님용 복습 노트

1. 그래프란 무엇인가

그래프는 정점(vertex)과 간선(edge)으로 이루어진 연결 구조입니다. 트리가 계층 구조를 표현하는 데 강하다면, 그래프는 더 자유로운 관계를 표현하는 데 강합니다.

기본 용어
  • vertex / node: 점, 대상 하나
  • edge: 두 정점을 잇는 연결
  • arc: 방향이 있는 간선이라고 생각하면 편함
  • path: 한 정점에서 다른 정점으로 가는 경로
  • cycle: 출발한 정점으로 다시 돌아오는 순환 경로
트리도 그래프의 한 종류입니다. 다만 트리는 사이클이 없고, 연결 구조가 깔끔하게 정리된 특별한 그래프입니다.

2. 그래프에서 먼저 구분해야 할 것

무방향 그래프

연결만 있고 방향은 없습니다.

1 -- 2 | 3

친구 관계, 양방향 도로 같은 느낌

방향 그래프

간선에 방향이 있습니다.

1 -> 2 1 -> 3 2 -> 4

선수과목, 작업 순서 같은 느낌

가중치 그래프

간선마다 거리, 비용, 시간 같은 숫자가 붙습니다. 이번 주 핵심은 BFS, DFS, 위상정렬이므로 가중치보다는 연결 구조와 탐색 방식에 먼저 익숙해지는 것이 중요합니다.

3. 그래프 표현법

초보 단계에서는 인접 리스트가 가장 중요합니다. BFS, DFS 문제에서 거의 기본처럼 쓰입니다.

graph = { 1: [2, 3], 2: [1, 4], 3: [1], 4: [2] }
  • graph[1] = [2, 3]은 1번 정점이 2, 3과 연결되었다는 뜻입니다.
  • 무방향 그래프라면 a - b를 저장할 때 양쪽에 모두 추가해야 합니다.
  • 방향 그래프라면 a -> b 한쪽만 추가합니다.
문제를 읽을 때 가장 먼저 볼 것: 정점 번호가 0부터 시작하는지, 1부터 시작하는지

4. BFS

BFS(Breadth-First Search)는 가까운 곳부터 차례대로 탐색하는 방식입니다. 한국어로는 너비 우선 탐색이라고 합니다.

핵심 감각
  • 시작점에서 1칸 떨어진 정점들을 먼저 본다
  • 그다음 2칸 떨어진 정점들을 본다
  • 레벨 순서대로 탐색한다고 생각하면 이해가 쉽다
A | \ B C | | D F

위 구조에서 A부터 BFS를 하면 보통 A -> B -> C -> D -> F 순서로 갑니다.

BFS는 보통 최단 거리 문제에서 강합니다. 가중치가 없을 때, 처음 도착한 거리가 가장 짧은 거리인 경우가 많기 때문입니다.
  • 주로 큐(queue)를 사용합니다.
  • 미로 탐색, 최소 이동 횟수, 단계별 전파 문제에서 자주 나옵니다.

5. DFS

DFS(Depth-First Search)는 한 방향으로 끝까지 깊게 들어갔다가, 더 못 가면 되돌아오는 탐색입니다. 한국어로는 깊이 우선 탐색이라고 합니다.

핵심 감각
  • 갈 수 있을 때까지 한 방향으로 내려간다
  • 막히면 돌아온다
  • 트리의 재귀 순회와 느낌이 비슷하다

DFS는 연결 요소 찾기, 경로 존재 여부 확인, 사이클 검사 같은 문제에서 매우 자주 쓰입니다.

  • 주로 재귀 또는 스택으로 구현합니다.
  • 순서는 인접 리스트 순서에 따라 달라질 수 있습니다.
BFS는 가까운 순서가 중요하고, DFS는 끝까지 파고드는 흐름이 중요합니다.

6. BFS와 DFS 비교

항목 BFS DFS
탐색 방식 가까운 곳부터 넓게 한 방향으로 깊게
주요 자료구조 큐 재귀 또는 스택
잘 맞는 문제 최단 거리, 레벨 탐색 경로 탐색, 연결 요소, 구조 확인
핵심 질문 가장 빨리 도착하려면? 연결되어 있는지 끝까지 따라가보자

7. 위상정렬

위상정렬(topological sort)은 순서가 정해져 있어야 하는 작업들을 앞뒤 관계를 지키며 나열하는 방법입니다.

예시
  • 양말을 신은 뒤 신발을 신어야 함
  • 선수과목 A를 수강한 뒤 과목 B를 들을 수 있음
  • 프로젝트에서 공통 모듈을 만든 뒤 기능 구현을 시작해야 함
A -> B A -> C B -> D C -> D

가능한 결과는 A, B, C, D 또는 A, C, B, D처럼 여러 개일 수 있습니다. 중요한 것은 선후관계를 깨지 않는 것입니다.

위상정렬은 사이클이 없는 방향 그래프에서만 가능합니다. 순환이 있으면 무엇을 먼저 해야 할지 결정할 수 없습니다.

8. 위상정렬에서 알아둘 개념

  • 진입차수(indegree): 어떤 정점으로 들어오는 간선 수
  • 진출차수(outdegree): 어떤 정점에서 나가는 간선 수

진입차수가 0이라는 것은, 그 정점을 시작하기 전에 먼저 해야 할 일이 없다는 뜻입니다. 그래서 위상정렬에서는 보통 진입차수가 0인 정점부터 처리합니다.

9. 문제를 풀 때 체크할 것

  • 방향 그래프인지 무방향 그래프인지
  • 정점 번호가 0부터인지 1부터인지
  • 방문 체크가 필요한지
  • 최단 거리 문제인지, 단순 연결 탐색 문제인지
  • 선후관계 정리 문제인지

이 다섯 가지만 먼저 구분해도 BFS, DFS, 위상정렬을 어느 방향으로 접근해야 할지 훨씬 빨리 보입니다.

10. 재혁님용 한 줄 정리

  • 그래프: 점과 선으로 연결 관계를 표현하는 구조
  • BFS: 가까운 곳부터 차례대로 탐색
  • DFS: 한 방향으로 깊게 들어갔다가 돌아오기
  • 위상정렬: 순서가 있는 작업들을 올바르게 나열하기
